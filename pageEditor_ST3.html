Team page

Technical Description

The Team page was developed using a structured HTML layout with clearly organised content to present team member profiles in a visually consistent format. Semantic elements such as header, nav, main, and footer were used to divide the page into meaningful sections, improving both readability and maintainability. Each team member is represented using a reusable div container with the class .student, which includes an image, name, ID, and interactive elements. Headings (h3) were used to display names, while p elements provide additional information such as roles and contact details.

CSS was used to create a modern card-based layout and enhance user interaction. Flexbox (display: flex) was applied to the .team container to align all student cards horizontally and distribute space evenly. Each card uses position: relative to allow layering of elements such as overlays and additional details. Images are styled using object-fit: cover and border-radius to maintain consistent proportions and a polished appearance. Hover effects were implemented using transform: scale() and transition to provide visual feedback when users interact with the cards.

An overlay effect was created using the ::after pseudo-element, adding a semi-transparent dark layer over each card to improve text visibility. Additional information is hidden initially using transform: translateY(100%) and revealed dynamically when a class is applied. This sliding panel effect enhances interactivity and keeps the interface clean.

JavaScript was used to implement dynamic functionality. Event listeners were attached to “More Details” buttons, allowing users to toggle the visibility of additional information by adding or removing a CSS class. Another event listener detects clicks outside the card to automatically close the details panel. Pointer events were temporarily disabled on social media icons when the details panel is open, preventing unintended interactions. These features improve usability by providing controlled and interactive content display.

Accessibility

Accessibility was considered to ensure the Team page can be used by a wide range of users. Semantic HTML elements such as header, nav, and main provide a clear structure, enabling screen readers to interpret the layout effectively. Headings are used appropriately to present names, helping users understand the content hierarchy.

All images include descriptive alt attributes, allowing users who rely on screen readers to understand the visual content. Links for social media and email are implemented using standard a elements, ensuring they are keyboard accessible and function correctly with assistive technologies.

Interactive elements such as buttons are implemented using button tags, making them accessible via keyboard navigation and ensuring proper focus behaviour. The use of consistent layout and spacing improves readability for all users. Additionally, hover and visual effects enhance usability without interfering with core functionality.

However, further improvements could be made to enhance accessibility. For example, adding aria-expanded attributes to the toggle buttons would help screen readers understand when additional content is visible. Providing more descriptive labels for social media icons and ensuring sufficient colour contrast would also improve usability for users with visual impairments.

Overall, the page combines semantic structure, interactive elements, and visual design to create an engaging and accessible user experience.

Feedback Page

Technical Description

The webpage was developed using a well-structured HTML layout with clearly organised sections, including navigation, program listings, reviews, and a feedback form. Semantic elements such as header, nav, main, and section were used to define different parts of the page, ensuring a logical and meaningful content structure. Headings (h1–h4) were applied to establish a clear hierarchy, making the content easier to read and understand. Lists (ul and li) were used within the navigation bar and program cards to organise links and actions efficiently.

CSS was used extensively to enhance the visual design and layout of the page. Flexbox (display: flex) was applied to align and distribute elements such as program cards and review sections, allowing for a responsive and flexible layout. Card components were styled using properties like border-radius, margin, and padding to create a clean and modern interface. Images were styled using object-fit: cover and object-position: center to ensure they scale correctly without distortion. Additionally, hover effects using transform: scale() and transition were implemented to provide interactive feedback when users engage with program cards.

The page also includes form styling to improve usability. Input elements (input, textarea, and select) were designed with consistent widths, padding, and rounded borders to maintain visual consistency. A scrollable container was created using max-height and overflow-y: auto, allowing users to browse program content without extending the page length excessively.

JavaScript was used to add dynamic functionality to the page. A character counter updates in real time using an input event listener, providing feedback on the remaining number of characters in the message field. Form validation was implemented using an event listener on form submission, ensuring required fields such as name and email are correctly filled before submission. Error and success messages are displayed dynamically, improving user interaction and preventing invalid data entry.

Accessibility

Accessibility was considered throughout the design to ensure the page is usable by a wide range of users, including those using assistive technologies. Semantic HTML elements such as header, nav, main, and section were used to create a meaningful structure, allowing screen readers to interpret and navigate the content effectively.

All images include descriptive alt attributes, ensuring that users who cannot see the images can still understand their purpose. Form inputs are clearly associated with label elements using the for attribute, improving usability for screen readers and enabling users to click on labels to focus input fields. The use of fieldset and legend further enhances accessibility by grouping related form controls and providing context.

The design maintains a clear visual hierarchy through the use of headings and consistent spacing, making content easier to follow. Interactive elements such as buttons and links are implemented using standard HTML elements, ensuring they are keyboard accessible. Additionally, validation messages provide immediate feedback, helping users understand and correct errors efficiently.

The page also includes responsive design features through the viewport meta tag, ensuring it adapts to different screen sizes and devices. Overall, the combination of semantic structure, labelled inputs, alternative text, and user feedback mechanisms contributes to a more inclusive and accessible user experience.

Content Page

Technical Description

The content page was developed using a structured HTML layout to organise information into a clear and readable format. Semantic elements such as header, nav, main, section, and aside were used to separate navigation, sidebar content, and the main article. The aside element was specifically used to create a sidebar navigation menu, allowing users to quickly access different topics and subtopics. Headings (h1 and h2) were used to establish a clear content hierarchy, making the page easier to scan and understand.

CSS was used to create a clean and functional layout. Flexbox (display: flex) was applied to the .layout container to position the sidebar and main content side by side. The sidebar was styled using position: sticky with top: 0, ensuring it remains visible as users scroll through the content. This improves navigation by keeping topic links accessible at all times. Spacing and alignment were controlled using padding, margin, and width properties to maintain a balanced layout.

Typography and visual styling were enhanced using consistent font sizes and colours, improving readability. Lists (ul and li) were styled to differentiate between main topics and subtopics using different list styles. A “Go to top” button was implemented using position: fixed, allowing users to quickly return to the top of the page regardless of their scroll position. This improves overall usability, especially for long content pages.

The design focuses on clarity and usability, ensuring that large amounts of text are presented in an organised and user-friendly way. The combination of structured HTML and CSS layout techniques results in a responsive and easy-to-navigate interface.

Accessibility

Accessibility was considered in the design of the content page to ensure it is usable by a wide range of users. Semantic HTML elements such as header, nav, main, section, and aside were used to provide a meaningful structure, allowing assistive technologies like screen readers to interpret the page effectively.

Headings (h1 and h2) are used in a logical order to create a clear content hierarchy, making it easier for users to navigate through the information. The sidebar navigation uses lists and links, which are easily accessible via keyboard and assistive technologies, enabling users to move between sections efficiently.

All images include descriptive alt attributes, ensuring that users who cannot see visual content can still understand its purpose. The “Go to top” button provides an additional navigation aid, helping users quickly return to the beginning of the page without excessive scrolling.

The layout is responsive due to the use of the viewport meta tag, ensuring usability across different screen sizes and devices. Text is presented with sufficient spacing and contrast to improve readability. Additionally, links are clearly distinguishable, helping users identify interactive elements easily.

To further improve accessibility, enhancements such as adding ARIA labels to navigation elements and ensuring strong colour contrast could be implemented. Overall, the page provides a structured, readable, and accessible experience for users with different needs.

Challenges and Lessons Learned

During the development of this page, several challenges were encountered in both design and functionality. One of the main challenges was creating a responsive and visually balanced layout using Flexbox. Aligning multiple card elements evenly across different screen sizes required careful adjustment of properties such as gap, width, and alignment settings. Ensuring that the layout remained consistent without breaking on smaller screens was particularly challenging.

Another difficulty was implementing the interactive card functionality. Managing the visibility of the hidden details section using CSS transformations (transform: translateY) and JavaScript required precise coordination between styling and scripting. Additionally, handling user interactions such as closing the details panel when clicking outside the card involved more advanced event handling logic, which required careful testing to avoid unexpected behaviour.

Working with layered elements using position: absolute and z-index also presented challenges. Ensuring that text, overlays, and interactive elements appeared in the correct order without interfering with each other required a clear understanding of stacking contexts. Similarly, controlling pointer events to disable social media icons when the details panel is open required additional logic to maintain usability.

From an accessibility perspective, it was challenging to balance visual design with usability. While the interactive and animated elements improved the user experience, it became clear that additional considerations such as ARIA attributes and keyboard interaction handling are necessary to make the page fully accessible.

Through this process, several key lessons were learned. Firstly, the importance of planning layout structure before styling became evident, as it simplifies development and reduces the need for major adjustments later. Secondly, integrating JavaScript with CSS for interactive features requires a clear understanding of how classes and styles interact. Finally, the experience highlighted the importance of accessibility considerations from the beginning of the design process, rather than treating them as an afterthought.

Overall, these challenges helped improve problem-solving skills and provided a deeper understanding of responsive design, DOM manipulation, and user-centered development.
