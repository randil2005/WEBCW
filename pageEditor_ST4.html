User Profile

Technical Description

This page uses semantic HTML like <header>, <nav>, <main>, and <footer> to organise content logically and improve accessibility. Navigation is structured with <ul> and <li> lists for clarity, while <a> tags enable internal linking between pages. Images use <img> with alt attributes for accessibility. CSS applies layout using Flexbox (display: flex) to create a responsive multi-section structure, and position: sticky keeps key elements like navigation visible. Styling such as padding, margins, and font properties enhances readability and consistent spacing.

Accessibility

Accessibility is supported through semantic tags like <header>, <nav>, and <main> for clear structure, along with alt text on <img> elements. Consistent layout and readable text styling improve navigation and usability.

Sitemap

Technical Description

The sitemap is built using nested <ul> and <li> elements to represent hierarchical navigation, making the structure clear and machine-readable. <a> tags link to all major pages, supporting usability and SEO. Minimal CSS is used, focusing on list styling (list-style-type) to visually differentiate levels, and spacing properties to maintain readability. The design prioritises functionality by providing a clear overview of site structure rather than complex styling.

Accessibility

Accessibility is enhanced using structured lists (<ul>, <li>) to clearly represent hierarchy, and descriptive <a> links for easy navigation. The simple layout ensures screen readers can interpret content effectively.

Content Page

Technical Description

This page uses a semantic layout with <section>, <article>, <figure>, and <figcaption> to group related content and media meaningfully. A Flexbox layout (display: flex) creates a sidebar and main content area, improving navigation and readability. The sidebar uses position: sticky for persistent navigation, while images are styled with width, margin, and alignment properties for responsiveness. Typography is controlled using font imports and text properties (text-align, font-size) to ensure a clean, readable design focused on content delivery.

Accessibility

Accessibility is achieved with semantic tags like <section> and <figure>, plus alt text and captions (<figcaption>) for images. The clear layout, readable typography, and logical content flow improve both visual and non-visual user experience.

Challenges and Lessons Learned

The main challenge faced during this group project was the consistency betweeen webpages and having to follow the same outline to ensure that the website looks neat throughout. Few issues were faced when css elements clashed between pages of the global css elements and the team was able to discuss and find alternate ways to achieve the same results.

User Profile Page

When it came to the user profile, all inputs were taken from the JavaScript prompt() tag which interprets all inputs as text. Since there is no defualt validation for these inputs, individual validations were created for data like mobile number and email using regular expressions.

Sitemap

Looking at the overall design of the sitemap, the biggest challenge was mapping coordinates in the svg plane. I considered it as a graphical plane which made positioning elements much easier and resulted in a clean and neat looking sitemap which gives access to every individual page linked to this website.

Compliance

User Profile Page

The document demonstrates a clean separation of concerns, with internal CSS and HTML structures working in tandem. The successful validation confirms that user-specific modules, including profile imagery and navigation links are properly nested, ensuring that the interface remains stable across various viewport sizes and devices.

Sitemap

The sitemap serves as the architectural backbone of the site. Validation confirms that the hierarchical list structures ul, li and anchor tags are syntactically perfect. This guarantees that search engine crawlers and assistive technologies can map the website's directory without encountering broken fragments or structural bottlenecks.

ST4 Content Page

Despite the complexity of this page, which includes multi-level navigation, semantic figures, and technical descriptions of energy systems the markup maintains a high degree of precision. The use of the UTF-8 character set and HTML5 structural elements ensures that the information is accessible and future-proofed against evolving web standards.

The results indicate a robust development workflow. By eliminating syntax errors and adhering to W3C standards, the website is optimized for SEO, Accessibility, and Performance. This technical foundation ensures a consistent and reliable experience for all end-users.
