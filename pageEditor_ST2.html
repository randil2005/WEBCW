Splash Screen

Technical Description

The splash screen was implemented using a structured HTML layout to present an introductory view of the website. The content is grouped within a <div> container with the class splashScreen, which contains headings, a logo image, team member details, a loader animation, and a countdown timer. This structure ensures that all elements are logically organised and easy to manage.

CSS was used to design and position the elements centrally on the page. Flexbox was applied to the <body> using display: flex, justify-content: center, and align-items: center to vertically and horizontally align the content. A background image was applied and styled using background-size: cover and background-position: center to ensure full-screen coverage across different devices.

The loader animation was implemented using CSS keyframes. A circular element was styled using borders, and the @keyframes rotate animation was applied to continuously rotate the loader, creating a smooth visual loading effect without the need for JavaScript.

JavaScript was used to implement a countdown timer. The setInterval() function decreases a timer value every second and dynamically updates the text content of the countdown element. Once the timer reaches zero, the interval is cleared, stopping the countdown. Additionally, a skip button was implemented using an onclick event, allowing users to immediately navigate to the home page using window.location.href.

Accessibility

Accessibility was considered in the design of the splash screen to ensure that it is usable for a wide range of users. Semantic HTML elements such as headings (<h1>, <h2>, and <h3>) were used to create a clear content hierarchy, allowing assistive technologies to interpret the structure of the page effectively.

The logo image includes descriptive alternative text, enabling screen reader users to understand its purpose. The skip button also includes an aria-label, providing additional context for users relying on assistive technologies and allowing them to bypass the splash screen and access the main content immediately.

Visual accessibility was addressed through the use of sufficient colour contrast between text and background, ensuring readability. The layout was designed to be simple and centred, reducing cognitive load and making the content easy to follow.

The countdown text is dynamically updated, providing clear feedback to users about when the page will transition. Additionally, the presence of a skip button ensures that users are not forced to wait, improving overall usability and accessibility.

AIS page

Technical Description

The Action Impact Simulator (AIS) was developed using a structured HTML layout to organise interactive elements clearly. The main content is contained within a <main> element, with two <section> elements (#section1 and #section2) used to group the action cards. Each action is represented using a <div> with the class actionCards, containing an image, heading, description, and impact points, ensuring a consistent and scalable structure.

CSS was used to design the layout and enhance visual interaction. Flexbox was applied to arrange the action cards horizontally using display: flex and spacing properties such as gap and margin. Hover effects and transformations were used to improve interactivity, while the .selected class visually highlights selected cards. The .cardsContainer element was styled to support dynamic background changes based on user interaction.

JavaScript was used to implement the core functionality of the simulator. All action cards were selected using document.querySelectorAll(), and event listeners were attached to detect user clicks. When a card is selected or deselected, the handleCardClick function updates the total score based on predefined impact values associated with each card.

Conditional statements were used to determine the user’s impact level (low, medium, or high) based on score thresholds. The page dynamically updates feedback using DOM manipulation, including updating the total score, displaying descriptive feedback, and changing the background image of the container. A visual indicator bar also changes colour based on the impact level, reinforcing the feedback provided to the user.

Accessibility

Accessibility was considered in the design of the Action Impact Simulator to ensure that interactive content is usable for a wide range of users. Semantic HTML elements such as <main>, <section>, and heading tags were used to provide a clear structure, allowing assistive technologies to interpret the content effectively.

All images within the action cards include descriptive alternative text, ensuring that users who rely on screen readers can understand the purpose of each action. Text content is organised using headings and paragraphs, improving readability and navigation throughout the page.

Visual feedback is provided through colour changes, background images, and highlighted selections, allowing users to easily understand their progress and impact level. The feedback summary section is hidden by default and only displayed when the user interacts with the simulator, reducing unnecessary content and improving focus.

The layout is designed to be simple and intuitive, with clearly identifiable interactive elements. Consistent styling, spacing, and sufficient colour contrast ensure that the content remains readable and accessible across different devices and screen sizes.

Content Page

Technical Description

The content page was structured using semantic HTML elements to organise information clearly and logically. The main layout is implemented using a <main> element with the class layout, which divides the page into two sections: a sidebar and the main content area. The sidebar is created using the <aside> element, while the main content is contained within a <section> element.

CSS Flexbox was used to design the layout, with the .layout class applying display: flex to position the sidebar and content side by side. The sidebar is styled using position: sticky with a top value of 0, allowing it to remain visible while the user scrolls through the page. This improves navigation and usability.

Navigation within the page was implemented using anchor links. Each section of the content, such as solar, wind, hydropower, and geothermal energy, is assigned a unique id attribute. These correspond to links in the sidebar, enabling users to quickly jump to specific sections of the page. Additional navigation links are provided to access other content pages, ensuring consistency across the website.

Images were included within each section to support the written content. CSS was used to ensure images are responsive by setting their width to 100% and height to auto, preventing distortion across different screen sizes. Overall, the page focuses on clear structure, readability, and efficient navigation.

A “Back to top” navigation feature was also implemented using an anchor link that references the top of the page. This allows users to quickly return to the beginning of the content without excessive scrolling, improving navigation efficiency.

Accessibility

Accessibility was considered in the design of the content page to ensure that information is accessible to a wide range of users. Semantic HTML elements such as <main>, <aside>, and heading tags were used to create a clear and meaningful structure, allowing assistive technologies to interpret the page effectively.

All images include descriptive alternative text, ensuring that users who rely on screen readers can understand the visual content. Headings are organised in a logical hierarchy, making it easier for users to navigate through different sections of the page.

The sidebar navigation provides clear and consistent links to both internal sections and other content pages. The use of anchor links allows users to quickly jump to relevant sections without excessive scrolling, improving overall usability.

The page design maintains sufficient colour contrast between text and background to ensure readability. The layout is simple and structured, reducing cognitive load and making the content easier to understand. Additionally, responsive design techniques ensure that the page remains accessible across different devices and screen sizes.

A “Back to top” link is provided at the bottom of the page, allowing users to easily navigate back to the top. This reduces the need for excessive scrolling and improves usability, particularly for users navigating long content sections.

Challenges and Lessons Learned

During the development of this project, several challenges were encountered, particularly when implementing interactive features using JavaScript. One of the main difficulties was managing the scoring logic within the Action Impact Simulator, ensuring that points were correctly added and removed when users selected or deselected actions. This required careful use of conditional statements and class toggling to maintain accurate state management.

Another challenge was ensuring proper HTML structure and validation. Initial errors such as incorrect nesting of elements and invalid CSS values were identified during validation and had to be corrected. This process improved understanding of how browsers interpret code and highlighted the importance of writing clean, well-structured HTML.

Designing a consistent and responsive layout across all pages was also challenging. Achieving proper alignment using CSS, particularly with Flexbox and positioning techniques such as position: sticky, required experimentation and adjustment to ensure elements displayed correctly on different screen sizes.

Through this project, several key lessons were learned. The importance of planning and structuring code before implementation became clear, as it helped reduce errors and improve efficiency. Additionally, the use of validation tools and debugging techniques proved essential in identifying and resolving issues.

Overall, this project improved understanding of HTML, CSS, and JavaScript integration, as well as the importance of accessibility, usability, and clean design in web development.
