Splash Screen

Technical Description

The splash screen was implemented using a structured HTML layout with clearly organised content sections. Semantic elements such as headings (<h1>, <h2>, and <h3>) were used to present the main theme of SDG 7 and supporting text, ensuring a logical content hierarchy. A <meta> refresh tag was included to automatically redirect users to the home page after four seconds, fulfilling the requirement for timed navigation.

CSS was used to enhance the visual presentation of the page. A full screen background image was applied using background-image, along with background-size: cover and background-position: center to ensure proper scaling across different screen sizes. Text was centred to improve readability and maintain a clean layout. A loading animation was created using CSS keyframe animation, where a circular element rotates continuously to provide visual feedback during the waiting period.

JavaScript was used to implement dynamic functionality, specifically a countdown timer that updates the displayed text every second using setInterval. This provides users with clear feedback on when the page will redirect. Additionally, an event listener was added to the skip button, allowing users to manually navigate to the home page using window.location.href. These features improve usability by offering both automatic and user controlled navigation options.

Accessibility

Accessibility was considered in the design of the splash screen to ensure usability for a wide range of users. Semantic HTML elements such as headings (<h1>, <h2>, and <h3>) were used to create a clear content hierarchy, allowing assistive technologies such as screen readers to interpret the page structure effectively.

Images on the page include descriptive alternative text, ensuring that users who cannot see visual content can still understand its purpose. Additionally, an aria-label was applied to the skip button to clearly describe its function, enabling screen reader users to bypass the splash screen and access the main content immediately.

The design also maintains sufficient colour contrast between text and background, improving readability. Providing both an automatic redirect and a manual skip option ensures flexibility, accommodating users with different needs and preferences.

AIS page

Technical Description

The Action Impact Simulator (AIS) was developed using a structured HTML layout to organise interactive content clearly. The main content is contained within a <main> element, with action cards grouped into two sections (<section id="section1"> and <section id="section2">) for logical organisation. Each action is represented using a <div> with the class actionCards, containing an image, heading, description, and associated impact points, ensuring consistency across all elements.

CSS was used to create a responsive and visually engaging layout. Flexbox was applied to arrange action cards using display: flex, justify-content, and spacing properties, ensuring even distribution across the page. Styling features such as hover effects, transformations, and box shadows were implemented to enhance interactivity. A container element (.cardsContainer) was styled to allow dynamic background image changes based on the user’s impact level, reinforcing visual feedback.

JavaScript was used to implement the core interactive functionality. Event listeners were attached to each action card to detect user clicks. When a card is selected, a selected class is toggled, and a total score is updated dynamically based on predefined impact values. Conditional statements are used to determine the user’s impact level (low, medium, or high) based on score thresholds.

The feedback system updates dynamically through DOM manipulation, including displaying the total score, updating descriptive text and modifying the background image. Additionally, a visual indicator is used to reinforce the user’s impact level through colour changes. The element changes colour dynamically based on the calculated score, providing a simple and immediate representation of the user’s overall impact. The feedback section is initially hidden and only appears when the user interacts with the simulator, improving usability and maintaining a clean interface.

Accessibility

Accessibility was considered in the design of the Action Impact Simulator to ensure that the content is usable for a wide range of users. Semantic HTML elements such as <main>, <section>, and headings were used to provide a clear and logical structure, allowing assistive technologies to interpret the page effectively.

All images within the action cards include descriptive alternative text, enabling users who rely on screen readers to understand the purpose of each action. Text content is clearly structured using headings and paragraphs, improving readability and navigation throughout the page.

Visual feedback is provided through colour changes, background images, and highlighted selections, helping users easily understand their progress and impact level. The feedback summary section is dynamically displayed only when interaction occurs, reducing unnecessary content and improving user focus.

The layout is designed to be simple and intuitive, with clearly identifiable interactive elements. Consistent styling and spacing improve readability, while the use of contrasting colours ensures that text remains visible against background elements.

Content Page

Technical Description

The content page was structured using semantic HTML elements to organise information clearly and logically. A <main> element with the class layout was used to divide the page into two main sections: a sidebar and the main content area. The sidebar was implemented using the <aside> element, containing a list of links to both internal sections and other content pages, supporting navigation across the website.

CSS Flexbox was used to create the layout, with the .layout class applying display: flex to position the sidebar and content side by side. The sidebar was styled using position: sticky to ensure it remains visible while scrolling, improving usability. Additional styling such as spacing, background colours, and typography were applied to maintain a clean and readable design.

Internal navigation was implemented using anchor links, allowing users to quickly navigate to specific sections of the page such as solar, wind, hydropower, and geothermal energy. Each section is identified using unique id attributes, which correspond to links in the sidebar. This enhances user experience by providing efficient access to different parts of the content.

Images were included within each section to visually support the written content. CSS was used to ensure images are responsive by setting their width to 100% and height to auto, preventing distortion across different screen sizes. Overall, the design focuses on clarity, readability, and ease of navigation.

Accessibility

Accessibility was considered in the design of the content page to ensure that information is accessible to a wide range of users. Semantic HTML elements such as <main>, <aside>, and heading tags were used to provide a clear and meaningful structure, allowing assistive technologies to interpret the content effectively.

All images include descriptive alternative text, ensuring that users who rely on screen readers can understand the visual content. Headings are used in a logical hierarchy, making it easier for users to navigate through different sections of the page.

The sidebar navigation provides clear and consistent links to both internal sections and other content pages, improving usability. The use of anchor links allows users to quickly jump to relevant sections without excessive scrolling.

Colour contrast between text and background was maintained to ensure readability, and the layout was designed to be simple and intuitive. The use of responsive design techniques ensures that the page remains accessible and usable across different screen sizes and devices.

Challenges and Lessons Learned

During the development of this project, several challenges were encountered, particularly when implementing interactive features using JavaScript. One of the main difficulties was managing the scoring logic within the Action Impact Simulator, ensuring that points were correctly added and removed when users selected or deselected actions. This required careful use of conditional statements and class toggling to maintain accurate state management.

Another challenge was ensuring proper HTML structure and validation. Initial errors such as incorrect nesting of elements and invalid CSS values were identified during validation and had to be corrected. This process improved understanding of how browsers interpret code and highlighted the importance of writing clean, well-structured HTML.

Designing a consistent and responsive layout across all pages was also challenging. Achieving proper alignment using CSS, particularly with Flexbox and positioning techniques such as position: sticky, required experimentation and adjustment to ensure elements displayed correctly on different screen sizes.

Through this project, several key lessons were learned. The importance of planning and structuring code before implementation became clear, as it helped reduce errors and improve efficiency. Additionally, the use of validation tools and debugging techniques proved essential in identifying and resolving issues.

Overall, this project improved understanding of HTML, CSS, and JavaScript integration, as well as the importance of accessibility, usability, and clean design in web development.
